## At a glance

- **What:** A small **GPT-2–style** model (decoder-only transformer) written **by hand in PyTorch**, then **loaded with real pretrained weights** so outputs match a reference implementation.
- **Why:** Almost every big LLM today still uses the **same core idea** (attention + feed-forward blocks, predict the next token). GPT-2 is the shortest path to seeing that clearly.
- **How to use this post:** Skim the bullets and diagrams first, then run the commands in **Run it locally** if you want to verify on your machine.

## Who this is for

## Why GPT-2 still matters

It is easy to dismiss GPT-2 as old news. In practice, **GPT-4-class models, Claude, Llama, and DeepSeek still share the same skeleton**: decoder-only transformer, causal self-attention, pre-norm residuals, and next-token prediction at the end. What changed is **scale, data, and engineering**—not a totally different architecture.

## What I built

A **from-scratch GPT-2 (124M)** in PyTorch: explicit `Linear`, `LayerNorm`, multi-head attention, GELU feed-forward, and **weight tying** on the language-model head (no `AutoModel` shortcut for the core forward pass).

Then I **loaded the real checkpoint**. That is the sanity check: if even one shape is wrong, weights do not line up or the logits are garbage. When generations look right, the code matches the math.

## Run it locally

**You need:** Python **3.10+**, `git`, and about **5–10 minutes** the first time (downloads weights).

1. Clone the repo and enter the code folder.
2. Create and activate a virtual environment (`python -m venv .venv`).
3. Install dependencies (`pip install -r requirements.txt`). Stack: **PyTorch**, **Hugging Face `transformers`** (for tokenizer/checkpoint loading only), **NumPy**.
4. Run generation or the optional attention visualization.

```bash
git clone https://github.com/saran-io/model-atlas.git
cd model-atlas/models/01-gpt2-from-scratch/code
python -m venv .venv
source .venv/bin/activate   # Windows: .venv\Scripts\activate
pip install -r requirements.txt
python gpt2.py --prompt "The future of AI is" --max_tokens 80
python gpt2.py --visualize --viz_text "The cat sat on the mat"
```

If a command fails, check that your venv is active and that PyTorch installed for your platform (see [pytorch.org](https://pytorch.org/get-started/locally/)).

## The stack (one screen)

Below is a **single-page view** of the forward pass: boxes are real HTML, styled for readability (including screen readers). No separate image file.

<Gpt2StackDiagram />

**How to read it, in one breath:** text becomes **token IDs**; IDs become **vectors** (token + position); then the **same block runs twelve times**—**attention** mixes information across positions, **feed-forward** refines each position, and **residuals** keep training stable. At the end, the **LM head** outputs scores over the vocabulary; weights are **tied** with the input embeddings (predicting token *k* and representing *k* share one space).

## Self-attention in one pass

The same pattern shows up in modern models (sometimes with extras like **GQA**, **RoPE**, etc.).

<SelfAttentionDiagram />

**Scaling (the 1/√d<sub>k</sub> step):** attention scores are divided by the **square root of the head dimension** so dot products stay in a range where **softmax** does not saturate. I removed that division once with correct weights loaded: **nonsense output**. One divisor is the difference between a toy and a working stack.
